## ⭐ 주요 개발 사항

### 1. 컴퍼스(Compass) 기능 구현
- **기능 상세 ([영상 참고자료 링크](https://www.notion.so/11b9957616c3801492bed08671944f17?pvs=21))**:
  - `HTMLCanvas`를 활용한 정밀 작도 기능.
  - 원/호 작도 및 작도된 도형의 선택, 이동, 삭제 지원.
  - 컴퍼스 모드 종료 시 하단 레이어 이벤트의 정상 동작 보장.
- **구현 방식**: `CanvasRenderingContext2D` 객체의 `arc`를 활용한 물리 드로잉 엔진 구축.
- **Troubleshooting**:
  - **문제**: 원을 그린 후 컴퍼스 모드를 종료해도 캔버스 레이어가 상단에 남아 있어 배경 버튼 클릭 등 이벤트 전달이 차단됨.
  - **해결**: 모드 종료 시 CSS `pointer-events: none`을 동적으로 부여하여, 시각적 요소는 유지하되 마우스 이벤트는 하단 레이어로 관통하도록 처리.

### 2. 정밀 타이머(Timer) 기능 구현
- **기능 상세 ([영상 참고자료 링크](https://www.notion.so/11b9957616c38018ba80e3bbec76fb08?pvs=21))**:
  - 지정한 시간만큼 정확하게 동작하는 타이머 도구 개발.
- **구현 방식**: `requestAnimationFrame`을 사용하여 브라우저 프레임 단위로 남은 시간을 계산하고 렌더링 최적화.
- **Troubleshooting**:
  - **문제**: `setInterval` 사용 시 단일 스레드 환경의 작업 부하로 인해 콜백이 지연되어 타이머 오차가 발생함.
  - **해결**: 화면 업데이트 주기에 맞춰 콜백을 실행하는 `requestAnimationFrame`과 타임스탬프 기반 계산 로직을 결합하여 정밀도 확보.
